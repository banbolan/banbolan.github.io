hexo 的写作

1.创建一个新的文章或者页面： hexo new [layout] <title>

  1. 布局Layout 这个在根目录的_config.yml文件中
    1. post 保存路径 : source/_posts
    2. page 保存路径: source
    3. draft 保存路径: source/_drafts
  2. 文件名称
    1. 默认以标题作为文件名称，但是可以输入 new_post_name参数改变默认的文件名称
    2. eg: :year-:month-:day-:title.md
    3. :title :标题，空格会被替换为短杠
    4. :year:年份，2018
    5. :mouth :月份，04
    6. :i_mouth: 月份，4
    7. :day :日期，07
    8. :i_day:日期，7
  3. 草稿
    1. 可以通过 --draft 查看草稿，eg： hexo server --draft
    2. 可以通过 publish发布草稿或者页面，将其移动到指定的文件夹下: hexo publish [layout] <title>
  4. 模板
  5. Hexo会根据 scaffolds 中的文件建立文件
  6. eg: hexo new photo "My Gallery"命令会寻找 photo.md文件

2. Front-matter

    ---
    title: Hello World
    date: 2013/7/13 20:46:25
    ---

上述代码为Front-matter的一个示例，用于指定个别文件的变量。

参数 描述 默认值
layout 布局
title 标题 文章的文件名
date 建立日期 文章的建立日期
updated 更新日期 文章的更新日期
comments 开启评论 true
tags 标签，不用于分页
categories 分类，不用于分页
permalinlk 覆盖文章的网址

  • 分类和标签

    • 分类有层次性，顺序性，标签没有层次和顺序
       categories:
   - Diary  ###注意这里的短横线之后有一个空格
   tags:
   - PS3
   - Games

4. 资源(Asset)引用

  1. 其代表除了 source 文件夹中除了文章之外的所有文件。例如图片，css，js文件等。如果hexo中只有少量的图片，可把他们放在 source/images 中。

  2. 文章资源文件夹：有规律的组织图片内容分布到各个文章上。首先在 _config.yml 将 post_asset_folder 选择为 true来打开。在资源文件管理功能打开之后，每一次使用 hexo new [layout] <title>命令时会创建一个新的文件夹命名与该文章文件一致，之后可通过相对路径引用该关联文件夹中的内容。

  3. 相对路径引用的标签插件

```
    &#123;% asset_path slug %&#125;
    &#123;% asset_img slug [title] %&#125;
    &#123;% asset_link slug [title] %&#125;
```

通过常规的markdown语法` ![](example.jpg)` 不会出现在首页上，但是可通过以下方法引用` &#123;% asset_img example.jpg This is an example image %&#125;`
